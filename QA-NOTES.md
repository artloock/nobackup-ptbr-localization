The current revision process included:

- UI text consistency
- mission and objective wording
- item and collectible naming
- system message clarity
- menu/settings terminology
- dialogue naturalness in PT-BR
- singular/plural ambiguity handling
- contextual adaptation of tactical/radio lines
- formatting awareness for spacing-sensitive strings

## In-Game QA Still Pending

Once access to the game/build is available, the following checks should be performed:

### 1. UI Display Validation
- text overflow
- clipping or cutoff issues
- alignment problems
- spacing-sensitive string behavior
- all-caps readability
- menu fit and button text visibility

### 2. Context Validation
- whether each line matches the actual gameplay situation
- whether singular/plural usage feels correct in live scenes
- whether radio lines sound appropriate in solo/co-op gameplay
- whether item names and prompts match on-screen function

### 3. Dialogue Flow Validation
- emotional timing
- subtitle readability
- sentence pacing
- consistency between speaker tone and scene context
- naturalness during high-pressure gameplay moments

### 4. Technical Validation
- special characters rendering correctly
- accent marks displaying properly
- punctuation preserved correctly
- no broken placeholders or formatting markers
- no errors caused by string length changes

### 5. Terminology Validation
- repeated gameplay terms appearing consistently
- UI/menu terminology matching final implementation
- mission and objective text remaining coherent across maps and phases
